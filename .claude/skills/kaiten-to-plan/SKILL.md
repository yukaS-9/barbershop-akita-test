# /kaiten-to-plan — Создание PLAN.md из карточки Kaiten

## Алгоритм

### 1. Запросить у пользователя пространство
Вызови `list_spaces` через MCP Kaiten и покажи пользователю список пространств. Спроси, в каком пространстве искать.

### 2. Запросить доску
Вызови `list_boards` с `space_id` выбранного пространства. Покажи пользователю список досок. Спроси, какую доску выбрать.

### 3. Запросить карточку
Вызови `list_cards` с `board_id` выбранной доски. Покажи пользователю список карточек (ID и название). Спроси, какую карточку взять (по ID).

### 4. Получить данные карточки
Вызови `get_card` с `card_id` выбранной карточки. Извлеки из ответа:
- `title` — название карточки
- `description` — описание с пошаговым планом
- `board.title` — название доски
- `owner.full_name` — владелец

Также получи колонки доски через `list_columns` с `board_id`.

### 5. Распарсить описание в шаги
Разбей `description` на шаги (нумерованные пункты 1., 2., 3....). Каждый пункт — отдельный шаг в чеклисте.

### 6. Создать или обновить PLAN.md
Проверь, существует ли файл `PLAN.md` в корне проекта:
- **Если нет** — создай с содержимым:
  - Заголовок: название карточки
  - Метаданные: пространство, доска, ID карточки, владелец, цель
  - Чеклист из шагов
  - Статус: "Всё в очереди"
- **Если есть** — перезапиши актуальными данными из карточки

### 7. Сообщить результат
Скажи пользователю, что `PLAN.md` создан/обновлён, и покажи сводку: сколько шагов, какая карточка.

## Формат PLAN.md

```markdown
# План работ — [Название карточки]

**Доска:** [название доски]
**Пространство:** [название пространства]
**Карточка:** [название карточки] (ID: [id])
**Владелец:** [owner]
**Цель:** [цель из описания]

---

## Пошаговый план

- [ ] **Шаг 1.** [пункт 1]
- [ ] **Шаг 2.** [пункт 2]
- [ ] **Шаг N.** [пункт N]

---

**Статус:** Всё в очереди
```
